[8] Alocação automática de platoons
    - Situação: o platoonAllocEngine já sugere quem vai a cada slot com base em
      critérios de prioridade (relic mínimo, menor GP do personagem). O próximo
      passo é transformar isso em alocação automática com confirmação — similar
      ao que o EchoBase faz via Comlink.
    - Ação: verificar se a API Comlink expõe endpoint para submeter alocação de
      platoon (POST /guild/platoon ou similar). Se disponível, adicionar botão
      "Alocar automaticamente" que envia as sugestões do engine direto ao jogo,
      com tela de confirmação antes de submeter.
    - Dependência: confirmar disponibilidade e formato do endpoint no Comlink.

[10] Tela inicial multi-evento (GAC / TW / ROTE)
    - Situação: o simulador foi construído para ROTE, mas a estrutura de roster,
      combatEngine e farmEngine pode ser reaproveitada para outros eventos.
    - Proposta: adicionar uma landing page antes da UI atual onde o jogador
      escolhe o modo:
        [ROTE]  → fluxo atual
        [TW]    → planejamento de defesa/ataque por zonas
        [GAC]   → sugestão de squads por divisão/liga
      A partir da seleção, carrega o conjunto de engines e UI do modo escolhido,
      mas compartilha roster, sync e persistência entre todos os modos.
    - Ação: criar tela de seleção em index.html (ou index-home.html separado).
      Implementar ROTE primeiro; TW e GAC como módulos futuros.
    - Prioridade: baixa — depende de estabilização completa do módulo ROTE.
